## Positioning for search

Significa is not a generic agency. The positioning must be reflected in every page's copy and metadata:

- **Not:** "We build digital products"
- **Yes:** "Significa is a product design and engineering studio that ships high-quality digital products and e-commerce stores for scale-ups in Europe and the US"

LLMs and search engines reward specificity. Vague claims are invisible.

## GEO content principles

Generative engines (ChatGPT, Claude, Perplexity, Google AI Overviews) prefer content that:

1. **States the claim directly in the first sentence.** Not "We believe great products start with great design" but "Significa is a product design and engineering studio based in Porto."
2. **Uses structured prose.** Short paragraphs. Clear headings. No walls of text.
3. **Cites specifics.** Numbers, sectors, named clients (where permitted), named methodologies.
4. **Answers questions.** Every service page must answer: what is it, who is it for, why Significa, what does the process look like.
5. **Has FAQ sections** on key commercial pages. Wrap with `FAQPage` schema.
6. **Avoids marketing language** that LLMs skip over: "craft", "bespoke", "tailor-made", "seamlessly", "leverage".

## Schema templates

All schemas validated against schema.org v29.4 (December 2025). Implement via `src/lib/seo.ts` — never inline in components.

**Critical notes:**
- `ProfessionalService` is **deprecated** in schema.org — do not use it.
- `LocalBusiness` is for brick-and-mortar businesses with public-facing physical locations. Significa operates globally online — use `Organization` as the base type.
- Use `@id` with hash URLs to link entities across pages rather than duplicating full objects.

### Organization (global — injected on every page via base layout)

```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "@id": "https://significa.co/#organization",
  "name": "Significa",
  "url": "https://significa.co",
  "logo": {
    "@type": "ImageObject",
    "url": "https://cdn.significa.co/brand/logo.png",
    "width": 200,
    "height": 200
  },
  "description": "Significa is a product design and engineering studio based in Porto, building high-quality digital products and e-commerce stores for scale-ups in Europe and the US.",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "Porto",
    "addressCountry": "PT"
  },
  "sameAs": [
    "https://www.linkedin.com/company/significadotco/",
    "https://github.com/significa",
    "https://twitter.com/significadotco",
    "https://instagram.com/significadotco/",
    "https://www.behance.net/significa",
    "https://www.youtube.com/@SignificaDotCo"
  ]
}
```

*Verify all social URLs before deploying.*

### FAQ (add to any commercial page with visible Q&A content)

Q&A content must be visible on the page — do not add FAQPage schema to hidden content.

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "[QUESTION]",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[ANSWER — plain text, factual, 2–4 sentences]"
      }
    }
  ]
}
```

## Link building priorities

1. Design awards with backlink potential: Awwwards, CSS Design Awards, Ecommerce London Awards
2. Thought leadership: Smashing Magazine, UX Collective, Medium
3. Client cross-promotion: co-written posts linking back to case studies
4. Broken backlink monitoring: quarterly in Ahrefs

External link rules:
- DA <40 → `rel="nofollow noopener"`
- DA ≥40 → `rel="noopener"` only
- Always `https` — never `http`

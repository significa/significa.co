## Our strategy.

### Systems architecture and planning

We start by making needs, objectives, constraints and risks explicit. Then we define the systems architecture, prioritise features and assign responsibilities to pick the right tech stack for the job.

### Security and data modelling

API contracts, data models and permission rules are set with security and privacy in mind from the start. Where legacy or third parties are involved, we add an integration layer to simplify, normalise data and protect performance.

### Performance targets

We set clear, practical targets for speed and scale — such as page load and API response times — and design to meet them, not just at launch but as the product grows.

### CI/CD and observability

Continuous integration, versioning and automated tests keep delivery predictable. Observability gives us the signals to act quickly when something needs attention.

### Design system to code

Design Systems translate to code as tokens and component libraries, keeping implementation tightly aligned with design decisions and making future changes faster and safer.

### Open source foundations

We contribute and rely on open source code to speed up and optimise our solutions. **Foundations** is our library of components, patterns and guides, built to be copied and adapted, speeding delivery while keeping accessibility and performance front and centre.
